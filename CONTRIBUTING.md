# Contributing to EchoVox

Thank you for your interest in contributing to EchoVox! We welcome contributions from everyone.

## Code of Conduct

Please be respectful and considerate of others. We aim to maintain a welcoming and inclusive community.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- A clear, descriptive title
- Steps to reproduce the issue
- Expected behavior vs actual behavior
- Screenshots if applicable
- Your environment (OS, browser, Docker version, etc.)

### Suggesting Features

Feature suggestions are welcome! Please open an issue with:
- A clear description of the feature
- Use cases and benefits
- Any implementation ideas you might have

### Pull Requests

1. **Fork the repository** and create a new branch from `main`
2. **Make your changes** following our coding standards
3. **Test your changes** thoroughly
4. **Update documentation** if needed
5. **Submit a pull request** with a clear description of changes

## Development Setup

See the [README.md](README.md) for setup instructions.

## Coding Standards

### Backend (Rails)
- Follow Ruby Style Guide
- Use meaningful variable and method names
- Add comments for complex logic
- Write tests for new features

### Python Service
- Follow PEP 8 style guide
- Use type hints where appropriate
- Add docstrings to functions
- Write tests for new endpoints

### Frontend (React)
- Follow Airbnb JavaScript Style Guide
- Use semantic HTML
- Ensure WCAG 2.1 AA accessibility compliance
- Add PropTypes or TypeScript types
- Write component tests

## Testing

Before submitting a PR:
- Run existing tests
- Add tests for new features
- Test manually in multiple browsers
- Test with keyboard navigation
- Test with a screen reader if possible
