# Accessibility Statement for EchoVox

## Our Commitment

EchoVox is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

## Conformance Status

EchoVox aims to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA. These guidelines explain how to make web content more accessible for people with disabilities and user-friendly for everyone.

## Accessibility Features

### Visual Accessibility
- ✅ **High Contrast**: Sufficient color contrast ratios throughout the interface
- ✅ **Resizable Text**: All text can be resized up to 200% without loss of functionality
- ✅ **Clear Focus Indicators**: Visible keyboard focus indicators on all interactive elements
- ✅ **Color Independence**: Information is not conveyed by color alone
- ✅ **Responsive Design**: Works on different screen sizes and zoom levels

### Motor Accessibility
- ✅ **Keyboard Navigation**: Full keyboard accessibility for all features
- ✅ **Large Touch Targets**: All interactive elements are at least 44x44 pixels
- ✅ **No Time Limits**: No time-based limitations on user actions
- ✅ **Skip Links**: "Skip to main content" link for quick navigation

### Cognitive Accessibility
- ✅ **Clear Language**: Simple, consistent language throughout
- ✅ **FALC Simplification**: Facile À Lire et à Comprendre text simplification
- ✅ **Pictograms**: Visual aids from ARASAAC to support understanding
- ✅ **Predictable Navigation**: Consistent navigation patterns
- ✅ **Error Prevention**: Clear error messages and validation

### Auditory Accessibility
- ✅ **Text-to-Speech**: Full Web Speech API integration
- ✅ **Adjustable Speech Settings**: Control speed, pitch, and volume
- ✅ **Visual Highlighting**: Text highlights during speech
- ✅ **No Audio-Only Content**: All content available in text form

### Screen Reader Support
- ✅ **Semantic HTML**: Proper heading hierarchy and landmarks
- ✅ **ARIA Labels**: Descriptive labels for all controls
- ✅ **Alt Text**: All images have appropriate alternative text
- ✅ **Form Labels**: All form inputs have associated labels
- ✅ **Status Messages**: Screen reader announcements for dynamic content

### Motion and Animation
- ✅ **Reduced Motion Support**: Respects `prefers-reduced-motion` setting
- ✅ **No Auto-Playing Content**: No automatic animations or sounds
- ✅ **User Control**: Users control all speech and animations

## Known Limitations

While we strive for full accessibility, some limitations exist:

1. **Pictogram Coverage**: ARASAAC pictograms may not be available for all words in all languages
2. **Voice Selection**: Available voices depend on the user's operating system and browser
3. **LLM Simplification**: Text simplification quality depends on OpenAI API availability (fallback simplification is available)
